# spec.md — 経費管理アプリ 仕様書

## 1. 目的・利用シーン

### 目的

紙のレシートや表計算ソフトで行っている個人の経費管理をブラウザ1枚に置き換える。OCRや承認ワークフローを除いた、業務用SaaSの核となるCRUD機能を最小構成で再現する。

### 利用シーン

- 出張・会食後にすぐレシートをアプリに入力しておき、月末の精算作業をまとめて行う
- カテゴリや金額で検索し、特定の経費をすばやく見つける
- 精算が完了した件をステータス更新して管理状況を把握する

### 対象ユーザー

ひとりで経費管理を行う個人（マルチユーザー・承認フローは対象外）

## 2. データ項目

### 経費1件の項目定義

| 項目名 | キー | 必須 | 型 | 備考 |
|---|---|---|---|---|
| ID | `id` | ○ | String | `Date.now()` ベースで自動採番 |
| 日付 | `date` | ○ | String | 経費発生日。形式: `YYYY-MM-DD` |
| カテゴリ | `category` | ○ | String | 下記の選択肢から選ぶ |
| 金額 | `amount` | ○ | Number | 円（整数）。0円は不可 |
| メモ | `memo` | — | String | 用途や訪問先など自由記述。空文字可 |
| ステータス | `status` | ○ | String | `pending` または `settled`。初期値は `pending` |
| 作成日時 | `createdAt` | ○ | String | ISO 8601 文字列。自動付与 |

### カテゴリの選択肢

`交通費` / `会議費` / `接待費` / `消耗品` / `通信費` / `その他`

### ステータスの定義

| 値 | 意味 | バッジ表示 |
|---|---|---|
| `pending` | 申請中（精算前） | 「申請中」アンバー系バッジ |
| `settled` | 精算済 | 「精算済」グリーン系バッジ |

## 3. 画面構成

### 全体レイアウト

PC表示専用のツーカラム構成。モバイル対応は対象外。

```
┌──────────────────────┬──────────────────────────────────────┐
│  左ペイン (380px固定)  │  右ペイン (flex-1)                    │
│                      │                                      │
│  [サマリカード]        │  [空状態 / 詳細 / フォーム] ← モード切替 │
│  [検索] [＋新規登録]   │                                      │
│  ─────────────────   │                                      │
│  [経費カード]          │                                      │
│  [経費カード]          │                                      │
│  [経費カード]          │                                      │
│  …                   │                                      │
└──────────────────────┴──────────────────────────────────────┘
```

### 左ペイン

| 要素 | 内容 |
|---|---|
| サマリカード | 合計金額・申請中件数・精算済件数を表示。一覧の変化に即時連動 |
| 検索ボックス | メモ・カテゴリを横断検索。入力のたびにリアルタイムで絞り込み |
| 新規登録ボタン | クリックで右ペインをフォームモードに切替 |
| 経費カードリスト | 作成日時の新しい順に縦並び。各カードに日付・金額・カテゴリ・メモ抜粋・ステータスバッジを表示 |
| 選択中カード | ハイライト表示（左ボーダーとアクセントカラー背景） |

### 右ペイン（3モード）

| モード | ID | 表示タイミング |
|---|---|---|
| 空状態 | `pane-empty` | 起動時・削除後・キャンセル後（何も選んでいない状態） |
| 詳細 | `pane-detail` | カードを選択したとき・保存後 |
| 登録・編集フォーム | `pane-form` | 「＋ 新規登録」または「編集」ボタンを押したとき |

#### 詳細モードに表示する要素

- 日付・カテゴリ・金額・メモ・ステータスバッジ
- 「編集」ボタン → フォームモードへ
- 「削除」ボタン → 確認ダイアログ → 空状態へ
- ステータス切替ボタン（「精算済にする」or「申請中に戻す」）

#### フォームモードに表示する要素

- 日付・カテゴリ（セレクト）・金額・メモの入力フィールド
- 「保存」ボタン（バリデーション後に登録または更新）
- 「キャンセル」ボタン（空状態または詳細に戻る）

## 4. CRUD 操作フロー

| 操作 | トリガー | 処理 | 結果 |
|---|---|---|---|
| **Create** | 「＋ 新規登録」→ フォーム入力 → 保存 | 新オブジェクト生成・localStorage保存 | 詳細モードへ遷移・カード追加・サマリ更新 |
| **Read（一覧）** | 起動時 | localStorageから全件読み込み | 左ペインにカード一覧表示 |
| **Read（検索）** | 検索ボックス入力 | メモ・カテゴリを部分一致で絞り込み | カードリストをリアルタイム更新 |
| **Update** | 詳細 → 「編集」→ フォーム編集 → 保存 | 既存レコードを上書き・localStorage保存 | 詳細モードへ遷移・カード更新・サマリ更新 |
| **Delete** | 詳細 → 「削除」→ 確認ダイアログOK | localStorageから該当レコード削除 | 空状態へ遷移・カード削除・サマリ更新 |
| **ステータス切替** | 詳細 → ステータス切替ボタン | ステータスを `pending` ↔ `settled` で反転・localStorage保存 | バッジ即時更新・サマリ即時更新 |

### バリデーション（フォーム保存時）

| 項目 | ルール |
|---|---|
| 日付 | 必須・有効な日付形式 |
| カテゴリ | 必須・選択肢の中から選択済み |
| 金額 | 必須・1以上の整数 |
| メモ | 任意（空文字可） |

## 5. 永続化方針

| 項目 | 内容 |
|---|---|
| 保存先 | ブラウザの `localStorage` |
| キー | `expense-records` |
| 保存形式 | 経費オブジェクトの配列を `JSON.stringify` した文字列 |
| 読み出し | `JSON.parse`。キーが未存在の場合は空配列 `[]` をデフォルト値とする |
| 書き込みタイミング | 作成・更新・削除・ステータス変更のたびに即時保存 |

サーバー・クラウドストレージ・外部APIは一切使用しない。データはブラウザ内にのみ存在する。

## 6. 初期データ

### 投入タイミング

アプリ起動時に `expense-records` キーが存在しない、または空配列の場合のみ、サンプルデータを自動投入する。2回目以降の起動では投入しない。

### サンプルデータの方針

- 件数: **5件**
- カテゴリはバラけさせる（全6カテゴリのうち5種を使用）
- 金額は少額〜中程度まで幅を持たせる
- `pending`（申請中）と `settled`（精算済）を混在させる
- 日付は現在日付から数日〜数週間前の範囲に設定する

### サンプルデータ例

| # | 日付 | カテゴリ | 金額 | メモ | ステータス |
|---|---|---|---|---|---|
| 1 | 直近 | 交通費 | 1,240円 | 東京〜新宿 往復 | pending |
| 2 | 数日前 | 会議費 | 3,800円 | 社内MTG コーヒー代 | settled |
| 3 | 1週間前 | 接待費 | 12,600円 | 〇〇社様 会食 | settled |
| 4 | 2週間前 | 消耗品 | 550円 | ボールペン・ノート | pending |
| 5 | 3週間前 | 通信費 | 2,200円 | モバイルWi-Fi 追加チャージ | pending |

## 7. 対象外の機能

今回のスコープには含めない。将来拡張の候補として列挙する。

| 機能 | 除外理由 |
|---|---|
| レシートOCR（画像からの自動入力） | 外部API・サーバー処理が必要 |
| 承認ワークフロー（申請者／承認者の役割分担） | マルチユーザー機能が前提 |
| クラウド保存・他端末共有 | サーバーサイドが必要 |
| ログイン・マルチユーザー | 認証基盤が必要 |
| 会計ソフト連携・仕訳出力 | 外部API連携が必要 |
| CSV / PDF出力 | 追加ライブラリまたはサーバーが必要 |
| モバイル対応（レスポンシブ） | PC表示のみで最小構成を優先 |
